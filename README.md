# Приложение h-matrix

Приложение для работы с матрицей Эйзенхауэра — инструментом приоритизации задач по критериям важности и срочности.

**Version**: 0.2.2

## Быстрый старт

1. Откройте `index.html` в современном браузере (Chrome, Edge)
2. Нажмите **Open** (Ctrl+O) и выберите workspace-папку с .md файлами задач
3. Просматривайте задачи в 4 квадрантах матрицы
4. Создавайте новые задачи через кнопку `+` в квадранте
5. Изменения сохраняются **автоматически** после каждого действия

## Горячие клавиши

| Клавиши | Действие |
|---------|----------|
| `Ctrl+O` | Открыть папку с задачами |
| `Ctrl+R` | Обновить задачи из папки |
| `Ctrl+S` | Сохранить вручную (опционально) |

## Интерфейс

**Toolbar:**
- Логотип **h-matrix** слева
- Путь к папке: `/имя-папки` (после открытия)
- **Dropdown "Вид"** — переключение между режимами: колонки, матрица, список
- Кнопка **Refresh** — обновить задачи из папки (Ctrl+R)
- Кнопка **Open** справа — открыть папку с задачами (Ctrl+O)

**Кнопка добавления задачи:**
- В режиме **матрицы** — кнопки `+` в заголовках квадрантов + счётчик задач справа
- В режиме **списка** — кнопки `+` в заголовках квадрантов + счётчик задач + большая круглая кнопка внизу справа (синяя, 64px)
- В режиме **колонок** — кнопки `+` в заголовках колонок + счётчик задач

**Примечание:** Кнопки Refresh и переключения вида активны только после открытия папки.
**Примечание:** Из-за ограничений безопасности браузеров папка выбирается явно при каждом запуске приложения.

**Режимы просмотра:**
- **Колонки** Column-view: (по умолчанию) — 4 вертикальные колонки с задачами по квадрантам
  - Колонки растянуты на всю ширину экрана
  - Splitters между колонками для изменения ширины
  - Вертикальный скролл в каждой колонке
  - Горизонтальный скролл при нехватке места
  - Цвет заголовка соответствует цвету квадранта в матрице
- **Матрица** Matrix-view: — 4 квадранта для визуальной приоритизации
  - Splitters между квадрантами для изменения размеров
- **Список** — List-view: все задачи сгруппированы по квадрантам в expandable блоки
  - Цвет заголовка блока соответствует цвету квадранта в матрице
  - Кнопка `+` в заголовке каждого квадранта для добавления задачи
  - Клик по заголовку сворачивает/разворачивает группу задач
  - Задачи внутри группы отсортированы по приоритету (P-1 сверху), затем по дате создания (старые сверху)
  - Кнопка добавления задачи внизу справа (синяя, круглая, 64px)
  - Квадранты растянуты по ширине без отступов, без закруглений

## Формат задач

**Важно:** Используется формат **1 задача = 1 файл**.

- Имя файла генерируется из имени задачи: пробелы заменяются на `-`, существующие `-` удваиваются (`--`)
- Пример: `"Задача с тире - в имени"` → `Zadacha-s-tire---v-imeni.md`
- Файлы с несколькими задачами игнорируются
- При переименовании файла имя задачи обновляется автоматически

Каждая задача имеет следующую структуру:

```markdown
# Название задачи
## Header
**срочность** - срочно/не срочно
**важность** - важно/не важно
**priority** - число (приоритет в квадранте)
**created** - YYYY-MM-DD HH:MM:SS (UTC, ISO 8601 без timezone)
**due-date** - YYYY-MM-DD HH:MM:SS (UTC, ISO 8601 без timezone, необязательно)

## body
Текст описания в формате markdown
```

**Важно:** Даты хранятся в UTC формате (ISO 8601), но отображаются в локальном времени пользователя.
Это позволяет переносить файлы между машинами в разных часовых поясах — время будет корректным.

**Пример:**
- Файл: `2026-05-04 12:00:00` (UTC)
- Отображение в Москве (UTC+3): `2026-05-04 15:00:00`
- Отображение в Нью-Йорке (UTC-4): `2026-05-04 08:00:00`

## Логика сохранения

- **1 задача = 1 файл** — каждая задача сохраняется в отдельный файл
- **Имя файла** генерируется из имени задачи (пробелы → `-`, `-` → `--`)
- **Автосохранение** — после создания, редактирования или удаления задачи
- **Переименование** — при изменении имени задачи файл переименовывается
- **Удаление файлов** — при удалении задачи файл удаляется из папки

## Особенности

- **3 режима просмотра** — колонки (по умолчанию), матрица (4 квадранта), список (группировка)
- **Splitters** — перетаскивайте границы между квадрантами/колонками для изменения размеров
- **Сортировка** — задачи в квадранте сортируются по приоритету (1 = важный) и дате создания
- **Блокировка** — до открытия папки интерфейс заблокирован (overlay с подсказкой)
- **File System Access API** — прямая работа с файлами на вашем компьютере
- **Автосохранение** — после каждого изменения (создание, редактирование, удаление)
- **Цветовая индикация** — дедлайны (красный/жёлтый/синий) и приоритеты (P-1/P-2-3/P-4+)
- **Tooltip для дедлайнов** — показывает время до/после дедлайна при наведении
- **Markdown в описании** — body задачи рендерится как markdown (списки, код, заголовки, ссылки)

## Поддержка Markdown

Описание задачи (`body`) поддерживает markdown-разметку при отображении в матрице и списке.

**Поддерживаемые элементы:**
- **Заголовки:** `# H1`, `## H2`, `### H3`
- **Жирный текст:** `**текст**`
- **Курсив:** `*текст*`
- **Код:** `` `inline` ``
- **Списки:** `- элемент` и `1. элемент`
- **Ссылки:** `[текст](url)`

**Пример:**
```markdown
## body
**Важная задача** с *описанием* и списком:
- Шаг 1
- Шаг 2

Ссылка: [документация](https://example.com)
```

**Примечание:** В окне редактирования body остаётся plain text — markdown рендерится только при просмотре.

## Технические детали

### Цветовая схема

Все цвета управляются через CSS-переменные в файле `css/variables.css`. Для изменения цветовой схемы достаточно отредактировать этот файл.

**Квадранты (Material Design — розовая/фиолетовая/циан палитра):**
- Квадрант 1 (Сделать сейчас) — розовый 600 (`#d81b60`, `--quadrant-1-bg`)
- Квадрант 2 (Запланировать) — фиолетовый 600 (`#8e24aa`, `--quadrant-2-bg`)
- Квадрант 3 (Делегировать) — циан 700 (`#0097a7`, `--quadrant-3-bg`)
- Квадрант 4 (Убрать) — тиловый 700 (`#00796b`, `--quadrant-4-bg`)

**Toolbar:**
- Фон — тёмно-серый (`#343a40`, `--toolbar-bg`)
- Граница — чёрный (`#212529`, `--toolbar-border`)

**Приоритеты:**
- P-1 (высокий) — бледно-красный (`#f8d7da`, `--priority-high-bg`)
- P-2—P-3 (средний) — бледно-оранжевый (`#fff3cd`, `--priority-medium-bg`)
- P-4+ (низкий) — бледно-синий (`#d1ecf1`, `--priority-low-bg`)

**Дедлайны:**
- Просрочен — красный Bootstrap (`#dc3545`, `--due-overdue-bg`)
- < 24 часов — жёлтый Bootstrap (`#ffc107`, `--due-urgent-bg`)
- > 1 дня — синий Bootstrap (`#0d6efd`, `--due-normal-bg`)

**Создано:**
- Дата создания — светло-синий (`#e7f1ff`, `--task-created-bg`)

### Ограничения платформы

- **File System Access API** — работает только в Chrome, Edge, Opera
- **1 задача = 1 файл** — файлы с несколькими задачами игнорируются при загрузке
- **Имя файла** генерируется из имени задачи автоматически (не хранится отдельно)

### Поведение приложения

**До открытия папки:**
- Интерфейс заблокирован (overlay с подсказкой)
- Кнопки View Toggle, Refresh, добавления задачи — неактивны

**После открытия папки:**
- Автосохранение после каждого изменения (создание, редактирование, удаление)
- Переименование задачи автоматически переименовывает файл
- Удаление задачи автоматически удаляет файл

Матрица Эйзенхауэра — это простой способ расставлять приоритеты задач по двум критериям: **важность** и **срочность**.

Состоит из 4 квадрантов:

По горизонтальной оси откладывается **срочность**

По вертикальной оси **важность**



                     МАТРИЦА ЭЙЗЕНХАУЭРА

                    Срочно                  Не срочно
              ┌────────────────────┬────────────────────┐
Важно         │ 1. Сделать сейчас  │ 2. Запланировать   │
              │                    │                    │
              │ - кризисы          │ - развитие         │
              │ - дедлайны         │ - обучение         │
              │ - срочные проблемы │ - важные проекты   │
              ├────────────────────┼────────────────────┤
Не важно      │ 3. Делегировать    │ 4. Убрать          │
              │                    │                    │
              │ - чужие мелкие     │ - пустая рутина    │
              │   срочные задачи   │ - отвлечения       │
              │ - часть звонков    │ - бессмысленный    │
              │ - часть писем      │   скроллинг        │
              └────────────────────┴────────────────────┘

1. **Срочные и важные** — сделать сразу  
   Пример: дедлайн сегодня, критическая ошибка, срочный звонок.

2. **Важные, но не срочные** — запланировать  
   Пример: обучение, спорт, стратегия, долгосрочный проект.

3. **Срочные, но не важные** — делегировать  
   Пример: часть рутинных запросов, мелкие организационные задачи.

4. **Не срочные и не важные** — убрать или отложить  
   Пример: бессмысленный скроллинг, отвлекающие мелочи.

**Зачем нужна:**
Она помогает не просто тушить срочные дела, а уделять время действительно важному — тому, что влияет на результат в долгую.

## UI/UX

### Column-view (по умолчанию)

На весь экран рисуется 4 вертикальные колонки по квадрантам.

**Элементы колонки:**
- **Заголовок** — название и иконка квадранта (цвет по типу)
- **Подзаголовок** — комбинация важности/срочности
- **Кнопка +** — добавить задачу (справа в заголовке)
- **Счётчик** — количество задач в колонке (справа от кнопки +)
- **Список задач** — карточки задач внутри колонки

**Сортировка задач:**
- По `priority` asc (P-1 самый важный, сверху)
- По `created` asc (старые сверху при одинаковом приоритете)

**Взаимодействие:**
- Клик по задаче — открыть окно редактирования
- Клик по кнопке + — создать новую задачу в этом квадранте
- Drag splitters — изменить ширину колонок
- Горизонтальный скролл — при нехватке места
- Вертикальный скролл — в каждой колонке отдельно

### Matrix-view

На весь экран рисуется матрица из 4 квадрантов.
Квадранты разделены splitters, так что размеры квадрантов можно менять.

**Элементы квадранта:**
- **Заголовок** — название и иконка квадранта (цвет по типу)
- **Кнопка +** — добавить задачу (справа в заголовке)
- **Счётчик** — количество задач в квадранте (справа от кнопки +)
- **Подзаголовок** — комбинация важности/срочности
- **Список задач** — карточки задач внутри квадранта

**Сортировка задач:**
- По `priority` asc (P-1 самый важный, сверху)
- По `created` asc (старые сверху при одинаковом приоритете)

**Формат карточки задачи:**
```
- **task-name** — Имя задачи жирным шрифтом
- **Due-date** — в одной строке с именем (справа), если указан
  - Красный фон — дедлайн прошёл
  - Жёлтый фон — осталось меньше 24 часов
  - Синий фон — больше 1 дня
  - **Tooltip** — при наведении: "Просрочена на X дн. Y ч." или "Осталось X дн. Y ч."
- **body** — текст описания в формате markdown (до 150 символов)
- **footer** — дата создания (синий фон) и приоритет (с цветом)
  - **Приоритет**: P-1 (бледно-красный), P-2—P-3 (бледно-оранжевый), P-4+ (бледно-синий)
```

**Взаимодействие:**
- Клик по задаче — открыть окно редактирования
- Клик по кнопке + — создать новую задачу в этом квадранте
- Drag splitters — изменить размер квадрантов

### List-view

Все задачи сгруппированы по квадрантам в expandable блоки.

**Структура:**
- **Заголовок квадранта** — цвет соответствует цвету в матрице:
  - Квадрант 1 (Сделать сейчас) — красный
  - Квадрант 2 (Запланировать) — синий
  - Квадрант 3 (Делегировать) — жёлтый
  - Квадрант 4 (Убрать) — серый
- **Кнопка +** — добавить задачу (справа в заголовке)
- **Счётчик задач** — количество задач в квадранте (справа от кнопки +)
- **Список задач** — внутри развёрнутого блока

**Формат задачи в списке:**
```
- **task-name** — имя задачи
- **Due-date** — справа от имени (с tooltip)
- **body** — краткое описание (до 200 символов)
- **footer** — дата создания и приоритет (с цветом)
```

**Взаимодействие:**
- Клик по заголовку квадранта — свернуть/развернуть группу
- Клик по кнопке + — создать новую задачу в этом квадранте
- Клик по задаче — открыть окно редактирования
- Пустые квадранты не отображаются

**Стиль:**
- Квадранты растянуты по всей ширине (без отступов по бокам)
- Без закруглений (как в matrix-view)
- Разделитель — горизонтальная линия между квадрантами


### Окно создания задачи

Окно выполнено в стиле **модального** окна Bootstrap.

**Title bar:**
- Название окна ("Новая задача" или "Редактирование: ...")
- Кнопка закрытия (справа)

**Main area (форма):**
- **task-name** * — поле ввода имени задачи (обязательно)
- **срочность** * — выпадающий список: срочно / не срочно
- **важность** * — выпадающий список: важно / не важно
- **priority** * — приоритет в квадранте (число, мин. 1)
- **due-date** — дедлайн (datetime-local, необязательно)
- **body** — описание задачи (markdown, необязательно)

**Footer bar:**
- **Закрыть** — закрыть без сохранения
- **Удалить** — удалить задачу (только при редактировании)
- **Сохранить** — сохранить и закрыть

**Валидация:**
- Имя задачи обязательно и должно быть уникальным
- При ошибке поле подсвечивается красным
- Сообщение об ошибке под полем имени

**Автосохранение:**
- После создания/редактирования/удаления задача автоматически сохраняется в файл
